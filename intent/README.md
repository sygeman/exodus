# Exodus Intent

Слой интерпретации смысла.

Он берет сырой ввод и превращает его в структурированное понимание задачи:
- цель
- ожидаемый результат
- ограничения
- допущения
- неопределенности
- возможные ошибки интерпретации

Intent не планирует и не решает. Он фиксирует смысл так, чтобы дальше не было "мы поняли не то".

**Поток данных:**
User input → Exodus Intent (что система поняла: цель, ограничения, допущения, неопределённости) → Exodus Control

**Ключевая идея:**
Система обязана сначала показать "я понял вот так" и "я собираюсь сделать вот так", и только после этого получает право выполнять.
