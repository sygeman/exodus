Exodus обеспечивает эволюцию продукта через контролируемые итерации, где намерение пользователя стабилизируется, решения централизуются, выполнение изолируется, а состояние остаётся консистентным без деградации между шагами.

Exodus UI
Единый интерфейс системы.
Он закрывает весь пользовательский цикл:
ввод исходной задачи
диалог для уточнения
показ того, что система поняла (Intent)
показ плана действий (Control)
подтверждение запуска
просмотр результата выполнения
наблюдение состояния системы и прогресса

Exodus Intent
Слой интерпретации смысла.
Он берет сырой ввод и превращает его в структурированное понимание задачи:
цель
ожидаемый результат
ограничения
допущения
неопределенности
возможные ошибки интерпретации
Intent не планирует и не решает. Он фиксирует смысл так, чтобы дальше не было “мы поняли не то”.
User input
→ Exodus Intent (что система поняла: цель, ограничения, допущения, неопределённости)
→ Exodus Control (что система собирается сделать: план, шаги, затрагиваемые части State, используемые skills)
→ подтверждение или правка пользователем
→ Exodus Execution (изолированное выполнение без интерпретации)
→ Exodus State (фиксация результата)
Ключевая идея:
Система обязана сначала показать “я понял вот так” и “я собираюсь сделать вот так”, и только после этого получает право выполнять.
Смысл:
ты переносишь контроль с уровня кода на уровень интерпретации, убирая основную причину ошибок — раннее и скрытое понимание задачи.

Exodus Control
Слой планирования и принятия решений.
Он берет Intent и формирует план выполнения:
декомпозиция задачи
выбор стратегии реализации
распределение работы
выбор инструментов и моделей из Infra
определение зависимостей шагов
Перед Execution Control обязан показать:
что система поняла
что она собирается делать
И дождаться подтверждения пользователя.

Exodus Execution
Изолированный слой выполнения задач.
Он:
получает snapshot из State
получает задачу от Control
запускает агента через ACP (opencode, qwencode и т.д.)
работает в чистом runtime (sandbox / container / process isolation)
ограничен skills, заданными Control
возвращает результат как изменения (diff / artifacts)
уничтожается или сбрасывается после выполнения
Execution не сохраняет память и не накапливает состояние.

Exodus State
Слой состояния и памяти продукта.
Он хранит:
текущую версию продукта
артефакты (код, конфиги, документы)
историю изменений
причины изменений
State отвечает за непрерывность. Он не дает системе “переписать всё заново и сломать эволюцию”.

Exodus Infra
Инфраструктура конечного продукта.
Определяет runtime, build, deploy и среду исполнения результата (web, desktop и т.д.), не влияя на логику разработки.
