Exodus обеспечивает эволюцию продукта через контролируемые итерации, где намерение пользователя стабилизируется, решения централизуются, выполнение изолируется, а состояние остаётся консистентным без деградации между шагами.

## Компоненты системы

- **[Exodus UI](./ui/README.md)** — Единый интерфейс системы
- **[Exodus Intent](./intent/README.md)** — Слой интерпретации смысла
- **[Exodus Control](./control/README.md)** — Слой планирования и принятия решений
- **[Exodus Execution](./execution/README.md)** — Изолированный слой выполнения задач
- **[Exodus State](./state/README.md)** — Слой состояния и памяти продукта
- **[Exodus Infra](./infra/README.md)** — Инфраструктура конечного продукта
