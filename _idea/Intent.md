# Exodus Intent

Слой интерпретации смысла и стабилизации намерений.

## Миссия

Превратить хаотичные мысли и обсуждения в чёткое, стабилизированное намерение, готовое к реализации.

## Два режима работы

### 1. Discovery Mode

Процесс исследования и накопления инсайдов:
- Создание тезисов (атомарных мыслей)
- Споры и аргументация
- Выявление связей между идеями
- Накопление контекста через диалог

**Выход:** Набор связанных тезисов, отражающих понимание проблемы.

### 2. Stabilization Mode

Процесс формирования чёткого намерения:
- Анализ тезисов из Discovery
- Формулирование Vision (цель, результат, ограничения)
- Уточнение через диалог
- Фиксация стабилизированного Intent

**Выход:** Структурированный Intent, готовый для Control.

## Поток данных

```
User Input
    ↓
Discovery Mode (тезисы, споры, инсайды)
    ↓
Stabilization Mode (Vision, ограничения, допущения)
    ↓
Stabilized Intent (фиксированное понимание)
    ↓
Exodus Control
```

## Ключевые принципы

1. **Discovery = Stabilization** — не отдельные этапы, а непрерывный процесс эволюции от хаоса к ясности
2. **Тезис как атом** — минимальная единица знания, простая мысль
3. **История важна** — можно проследить "как мы к этому пришли"
4. **Проверка понимания** — система обязана показать "я понял так" перед переходом к Delivery

## Интеграция

- **Вход:** Пользователь через UI/Chat
- **Выход:** Stabilized Intent → Control Service
- **Хранение:** База данных для тезисов, Git для артефактов Delivery
