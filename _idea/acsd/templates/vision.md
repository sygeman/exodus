# System Vision

## 1. Purpose

Краткое описание зачем существует система.

Формат:
Система <имя> существует для того, чтобы <решаемая проблема> для <тип пользователя>.

---

## 2. Core Value

Какая ключевая ценность создаётся.

- основная ценность
- что делает систему уникальной
- почему она лучше альтернатив

---

## 3. System Scope

Что система **включает**.

- основной домен
- ключевые функции
- тип данных/операций

Что система **НЕ включает**.

- явно запрещённые области
- что должно решаться внешними системами

---

## 4. Primary Actors

Кто взаимодействует с системой.

| actor | описание |
|------|----------|
| user | конечный пользователь |
| system | внешняя система |
| admin | управление |

---

## 5. Core Capabilities

Список **возможностей**, не реализации.

- capability 1
- capability 2
- capability 3

Важно: capability = *что возможно*, а не *как это сделано*.

---

## 6. System Invariants

Инварианты системы.  
То, что **никогда не должно нарушаться**.

Примеры:

- данные пользователя принадлежат пользователю
- операции должны быть детерминированными
- интерфейсы модулей должны быть стабильными


---

## 7. Architectural Principles

Принципы архитектуры.

Например:

- модульность
- отсутствие циклических зависимостей
- контрактно-ориентированные интерфейсы
- декларативная спецификация

---

## 8. Non-Goals

Что система **не пытается решить**.

Это важно, чтобы AI не расширял scope.

Примеры:

- система не является CMS
- система не реализует собственную БД
- система не содержит бизнес-аналитику

---

## 9. Success Criteria

Как понять, что система реализована.

Примеры:

- все модули имеют формальный интерфейс
- контрактные тесты покрывают интерфейсы
- архитектурный граф не содержит циклов
