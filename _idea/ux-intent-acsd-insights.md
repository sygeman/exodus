# UX+Intent+ACSD Integration Insights

## Ключевые инсайты из дискуссии

### 1. Граф-центричность
Визуализация ACSD — база интерфейса. Всё вращается вокруг графа, чат — вторичный инструмент работы с графом.

### 2. Stateless UI
Нет сессий, нет истории чатов. Каждое взаимодействие — чистый запрос с контекстом, собранным из текущего состояния графа.

### 3. Smart Context
Контекст для LLM формируется автоматически из ACSD-графа: выбранная нода + соседи + связи + gaps. Не хранится, а собирается на лету.

### 4. Фрагментированный каскад как норма
Проекты могут загружаться с любым уровнем заполненности. Система показывает gaps и предлагает действия по заполнению.

### 5. Единый поток Intent→ACSD
Discovery/Stabilization не отдельный режим, а работа с L0 Vision. После стабилизации Vision автоматически появляется в графе.

## Открытые вопросы для принятия архитектурных решений

### A. Границы контекста
- Какой глубины собирать контекст из графа? (только соседи / полная трассировка / настраиваемо)

### B. "Память" отказов
- Если пользователь отклонил предложение LLM — запоминать это в графе или каждый раз предлагать заново?

### C. Визуализация gaps
- Показывать [?] пустые ноды прямо в графе или отдельной панелью?

### D. Режимы vs Контекст
- Явные режимы (Discovery/ACSD) или контекст определяется выбором в графе?

### E. Undo/Redo
- Через git историю или UI хранит стек операций?

## Принципы работы

1. **Клик по ноде** → Показывает детали, собирает контекст
2. **Команда в чате** → Выполняет операцию над графом
3. **Граф обновляется** → Визуально показывает изменения
4. **Контекст сбрасывается** → Готов к следующему взаимодействию
