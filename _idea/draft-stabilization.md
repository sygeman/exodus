# Draft → Stabilization: ACSD из существующего проекта

## Проблема

Сейчас ACSD предполагает создание каскада сверху вниз (L0 → L4). Но что если у нас уже есть проект без ACSD? Как "документировать" его задним числом?

## Идея

Загружаем существующий проект → система создаёт draft-элементы из всего что есть → потом пытаемся стабилизировать: вывести уровни, связи, восстановить ACSD.

## Поток

```
Существующий проект (без ACSD)
         ↓
Парсинг кода и файлов
         ↓
Draft-элементы (level = null)
  - файл: auth.ts
  - функция: login()
  - модуль: database
  - readme: "приложение для заметок"
  - и т.д.
         ↓
Анализ связей
  - login() → auth.ts (contains)
  - auth.ts → database (uses)
  - readme → описывает всё
         ↓
Попытка стабилизации (LLM)
  - предлагает уровни для каждого элемента
  - "readme" → L0 Vision
  - "auth.ts" → L2 Architecture
  - "login()" → L3 Component
  - связи подтверждаются/правятся
         ↓
Stabilized ACSD
```

## Модель элемента

```
CascadeElement:
  id: uuid
  content: "любой контент"  # код, текст, схема
  level: null | L0 | L1 | L2 | L3 | L4  # изначально null
  relations: []  # связи с другими элементами
  status: draft | stabilized
```

## Ключевые принципы

1. **Уровень — это вывод, а не ввод** — пользователь не гадает "это L2 или L3?", система предлагает
2. **Связи первичны** — важнее как элементы связаны, чем на каком они уровне
3. **Нет принуждения к структуре** — кидаешь что думаешь/видишь, система разберётся
4. **Постепенная стабилизация** — можно частями разбирать и подтверждать

## Что нужно реализовать

1. **Парсер проекта** — извлекает элементы (файлы, функции, зависимости, документацию)
2. **Draft хранилище** — элементы без уровней, привязанные к проекту
3. **LLM анализатор** — предлагает уровни и связи для draft-элементов
4. **UI стабилизации** — пользователь подтверждает/правит что система предложила

## Преимущества

- **Legacy → ACSD** — любой существующий проект можно "документировать"
- **Не с нуля** — не надо придумывать vision, он выводится из того что уже есть
- **Intent растворяется в Cascade** — нет отдельного слоя, draft-элементы и есть начало каскада
- **Обратный инжиниринг намерения** — система понимает что ты хотел, глядя на код
